Cheat Sheet CHAPTER 1 · INTRODUCTION TO OS
🖥What is an OS
›
Definition: Intermediary between user and hardware — manages resources & enables program execution
›
Resource Allocator: Manages all resources; decides between conflicting requests for fair, efficient use
›
Control Program: Controls execution to prevent errors and improper use
›
Kernel: The ONE program always running; everything else is a system or application program
›
No universal definition — varies wildly between vendors
🔲Computer System Structure
›
4 Components: Hardware → OS → Application Programs → Users
›
Hardware: CPU, memory, I/O devices — basic computing resources
›
OS: Controls & coordinates hardware use among apps and users
›
Users: People, machines, other computers
›
Bootstrap: Stored in ROM/EPROM (firmware); initializes system & loads kernel at power-up
⚡Interrupts
›
Interrupt: Device controller signals CPU it's done by causing an interrupt
›
Interrupt Vector: Contains addresses of ALL interrupt service routines
›
Trap/Exception: Software-generated interrupt (e.g., divide by zero, system call)
›
OS saves: Registers & program counter when interrupt occurs
›
OS is interrupt-driven — polling vs. vectored interrupt handling
📡I/O Structure & DMA
›
Synchronous I/O: Control returns to program only AFTER I/O completes
›
Async I/O: Control returns immediately without waiting
›
DMA: Device controller transfers blocks directly to RAM without CPU — only 1 interrupt per block
›
Device-status table: Entry per device: type, address, state
›
Each controller: Manages its device type; has a local buffer
💾Storage Hierarchy
›
Fastest → Slowest: Registers → Cache → RAM → SSD → HDD → Optical/Tape
›
Main Memory (RAM): Only storage CPU accesses directly; volatile; random access
›
Secondary Storage: Large, non-volatile (HDD: tracks→sectors; SSD: faster, non-volatile)
›
Tertiary: Optical, tape — WORM or RW
›
Device Driver: Uniform interface between controller and kernel
⚡Caching
›
Caching: Copying info into faster storage temporarily
›
Cache hit: Data found → used directly (fast)
›
Cache miss: Data fetched from slower storage, copied to cache, then used
›
Cache < storage it caches → replacement policy is a key design problem
›
Multiprocessor: Cache coherency — all CPUs must see the most recent value
🔧System Architecture
›
SMP: Symmetric Multiprocessing — all CPUs perform all tasks
›
AMP: Asymmetric — each CPU assigned a specific task
›
Multi-core: Multiple cores on one chip — faster on-chip communication
›
Clustered: Multiple systems sharing storage via SAN; high availability
›
Asymmetric cluster: Hot-standby mode | Symmetric: mutual monitoring
📋OS Structure & Multiprogramming
›
Multiprogramming: Keeps CPU busy by switching jobs when one waits for I/O
›
Timesharing/Multitasking: CPU switches so fast users can interact — goal <1 sec response
›
Virtual Memory: Run processes not fully in RAM
›
Swapping: Move processes in/out of memory
›
Job Scheduling: Select next job to run from memory subset
🔐Dual-Mode & OS Operations
›
Dual-Mode: User mode vs Kernel mode — distinguished by hardware mode bit
›
Privileged instructions: Only in kernel mode
›
System call: Switches to kernel mode; return resets to user mode
›
Timer: Prevents infinite loops — counter decremented by clock; interrupt at zero
›
VMM mode: Supports guest VMs on modern CPUs
▶Process Management
›
Process: Program in execution — active entity (program = passive)
›
Needs: CPU, memory, I/O, files, initialization data
›
Single-threaded: One program counter | Multi-threaded: one PC per thread
›
OS activities: Create/delete, suspend/resume, synchronization, communication, deadlock handling
›
Concurrency: Achieved by multiplexing CPUs among processes/threads
🗃Memory & Storage Management
›
Memory Mgmt: Determines what's in memory and when — tracks usage, allocates/deallocates
›
Storage abstraction: Physical storage → logical unit = FILE
›
File system: Create/delete files & dirs, access control, backup, map to secondary storage
›
Mass-storage: Free-space mgmt, storage allocation, disk scheduling
›
I/O subsystem: Hides hardware quirks — buffering, caching, spooling, device drivers
🛡Protection & Security
›
Protection: Controls access of processes/users to OS-defined resources
›
Security: Defense against attacks — DoS, worms, viruses, identity theft
›
User ID: Name + number, one per user — associated with all their files/processes
›
Group ID: Defines a set of users with shared access controls
›
Privilege escalation: Change to effective ID with more rights
🧩Kernel Data Structures
›
Linked lists: Singly, doubly, circular
›
BST: left ≤ parent ≤ right; O(n) search; balanced BST: O(log n)
›
Hash map: Hash function → fast key-value lookup
›
Bitmap: n binary digits representing status of n items (e.g., free/used)
›
Linux headers: <linux/list.h>, <linux/kfifo.h>, <linux/rbtree.h>
🌐Computing Environments
›
Mobile: iOS & Android; GPS, gyroscope, AR; optimized for battery
›
Distributed: Heterogeneous systems via TCP/IP (LAN/WAN/MAN/PAN)
›
Client-Server: Compute-server (services) / File-server (storage)
›
P2P: No client/server distinction — all nodes are peers (Napster, Skype)
›
Cloud: SaaS / PaaS / IaaS; Public / Private / Hybrid; built on virtualization
›
Real-time: Fixed time constraints — correct = accurate + on time
💻Virtualization & Open-Source OS
›
Virtualization: Guest OS runs within a host OS via VMM
›
Emulation: Source CPU type ≠ target CPU (slowest method)
›
VMM examples: VMware ESX, Citrix XenServer (no general-purpose host)
›
Open-Source OS: Source code publicly available — counter to DRM/closed-source
›
GPL (Copyleft): FSF license — derivatives must stay open-source. Examples: GNU/Linux, BSD UNIX
📏Storage Units Quick Reference
›
Bit: Basic unit — 0 or 1
›
Byte: 8 bits (smallest convenient chunk)
›
KB / MB / GB: 1,024 / 1,024² / 1,024³ bytes
›
TB / PB: 1,024⁴ / 1,024⁵ bytes
›
Networking: Measured in BITS (not bytes) — networks move data a bit at a time
